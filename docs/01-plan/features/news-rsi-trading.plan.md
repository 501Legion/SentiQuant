## 1. 개요

### 1.1 목적

뉴스 감성 분석(NLP)과 RSI 기술 지표를 결합하여:
- 매일 장 마감 후 신호(Strong Buy / Buy / Neutral / Sell / Strong Sell)를 자동 계산
- 다음 날 시가(Open) 기준으로 가상 매수/매도를 시뮬레이션
- 포트폴리오 수익률을 추적하여 전략 효과를 검증

### 1.2 대상 시장

- **시장**: 미국 주식 (NYSE, NASDAQ)
- **운영 기준 시간**: 미국 동부 시간 (ET) 정규장 09:30~16:00 기준
- **실행 타이밍**:
  - **신호 계산**: 매일 장 마감 후 (16:30 ET 이후) 실행
  - **가상 주문**: 다음 날 시가(Open) 데이터 수신 후 체결 처리

## 2. 요구사항

### 2.1 기능 요구사항

#### FR-01. 데이터 수집 모듈

| ID | 요구사항 |
|----|----------|
| FR-01-1 | Polygon.io API로 대상 종목의 과거 60거래일(~2개월) 일별 OHLCV 수집 |
| FR-01-2 | NewsAPI로 종목 관련 최근 7일치 영어 뉴스(제목, 요약, 발행일) 수집 |
| FR-01-3 | 뉴스는 정식 언론사 기사만 대상 (SNS, 블로그 제외) |
| FR-01-4 | API 호출 실패 시 재시도 로직 및 에러 로깅 |

#### FR-02. 지표 계산 모듈

| ID | 요구사항 |
|----|----------|
| FR-02-1 | RSI(14일) 계산: `RSI = 100 - (100 / (1 + RS))`, RS = avg_gain / avg_loss |
| FR-02-2 | RSI MA(7일): RSI의 7일 단순이동평균 |
| FR-02-3 | 감성 점수: TextBlob Polarity 평균 → `Scaled Sentiment = (avg_polarity + 1) × 50` → [0, 100] |
| FR-02-4 | 7일 치 뉴스 전체 평균 극성 사용 |

#### FR-03. 신호 생성 모듈 (Decision Engine)

| 신호 | 감성 점수 조건 | RSI 조건 |
|------|--------------|---------|
| Strong Buy | > 70 | < 30 |
| Buy | > 50 | 30 ≤ RSI < 50 |
| Neutral | 40 ~ 60 | 40 ~ 60 |
| Sell | < 50 | > 70 |
| Strong Sell | < 30 | > 70 |

**주의**: 위 조건에 해당하지 않는 경우 가장 근접한 신호 또는 Neutral 반환 (신호 없음 방지)

#### FR-04. 페이퍼 트레이딩 엔진

| ID | 요구사항 |
|----|----------|
| FR-04-1 | 초기 가상 자금 설정 (기본값: $100,000) |
| FR-04-2 | **매수 규칙**: Buy/Strong Buy 신호 → 다음 날 시가에 가상 매수 |
| FR-04-3 | **추가 매수**: 연속 Buy 신호 시, 다음 날 시가 < 이전 매수 평균가일 때만 추가 매수 |
| FR-04-4 | **매도 규칙**: Neutral/Sell/Strong Sell 신호 → 다음 날 시가에 가상 매도 |
| FR-04-5 | **Neutral 매도 조건**: 순수익률 > 1%일 때만 매도 (`(sell - avg_buy) / avg_buy × 100 > 1`) |
| FR-04-6 | **보유 기간 조정**: 14일 경과 + 순수익 < 1% → 목표 수익률 0.25%로 하향 후 조건 충족 시 매도 |
| FR-04-7 | 포지션별 평균 매수가, 보유 수량, 보유 기간 추적 |

#### FR-05. 스케줄러 모듈

| ID | 요구사항 |
|----|----------|
| FR-05-1 | 미국 정규장 스케줄 인식 (NYSE 휴장일 제외) |
| FR-05-2 | 장 마감 후 (16:30 ET) 신호 계산 자동 실행 |
| FR-05-3 | 다음 날 시가 확정 후 (09:35 ET) 가상 주문 처리 |
| FR-05-4 | 코드 실행 중 지속 동작 (백그라운드 루프 또는 스케줄러) |

#### FR-06. 포트폴리오 추적 및 리포트

| ID | 요구사항 |
|----|----------|
| FR-06-1 | 현재 보유 종목, 수량, 평균 매수가, 평가 손익 표시 |
| FR-06-2 | 거래 이력 저장 (종목, 날짜, 신호, 체결가, 수량, 손익) |

## 3. 시스템 아키텍처 개요

```
[Scheduler]
    │
    ├── 16:30 ET: Signal Calculation Job
    │       ├── [Data Collector] → Polygon.io (OHLCV)
    │       ├── [Data Collector] → NewsAPI (News)
    │       ├── [Indicator Engine] → RSI(14) + RSI MA(7)
    │       ├── [Sentiment Engine] → TextBlob → Scaled Score
    │       └── [Decision Engine] → Signal (5-level)
    │
    └── 09:35 ET: Order Processing Job
            ├── [Data Collector] → Next day Open price
            ├── [Paper Trading Engine] → Virtual Buy/Sell
            └── [Portfolio Tracker] → Update + Report
```

## 4. 기술 스택

| 구성요소 | 기술 |
|---------|------|
| 언어 | Python 3.10+ |
| 데이터 수집 | `requests`, `polygon-api-client` |
| 뉴스 수집 | `newsapi-python` |

| 스케줄러 | `APScheduler` 또는 `schedule` |

| 장 달력 | `pandas_market_calendars` (NYSE 휴장일) |
| 데이터 저장 | JSON / CSV (SQLite 선택 가능) |
| 설정 관리 | `python-dotenv` |

## 5. 리스크

| 리스크 | 영향도 | 대응 방안 |
|--------|--------|-----------|
| NewsAPI 무료 티어 제한 (100req/day, 1개월 이내 기사만) | 중 | 종목 수 제한, 유료 전환 고려 |
| Polygon.io 무료 플랜 API 속도 제한 | 중 | 요청 간 딜레이 추가, 캐싱 |
| 다음 날 시가 데이터 수신 지연 | 중 | 재시도 로직 + 시가 확정 대기 시간 여유 |
| 신호 조건 미해당 케이스 (엣지 케이스) | 중 | 조건 미해당 시 Neutral 기본값 처리 |
| TextBlob 영어 외 텍스트 처리 오류 | 낮음 | 언어 필터링 (영어 기사만) |
| 스케줄러 프로세스 중단 | 중 | 로그 + 재시작 가이드 문서화 |
